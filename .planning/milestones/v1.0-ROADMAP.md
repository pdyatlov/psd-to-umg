## Overview

PSD2UMG v1.0 delivers a production-grade Unreal Engine 5.7 editor plugin that imports `.psd` files and converts them into fully functional UMG Widget Blueprints in one click. Starting from a UE4/Python-based fork (AutoPSDUI), the entire import pipeline was rewritten in C++20, integrating the PhotoshopAPI native library, a pluggable layer-mapper architecture, full text/typography support, layer effects, advanced layout features, editor UI, CommonUI interop, comprehensive testing, and a unified `@`-tag-based layer naming system.

**Core Value delivered:** A designer drops a PSD into Unreal Editor and gets a correctly structured, immediately usable Widget Blueprint — with no Python dependency, no manual tweaking, and no loss of layer intent.

## Key Accomplishments

1. **Native C++ PSD parser replacing Python pipeline (Phase 2)** — PhotoshopAPI integrated as pre-built static lib; FPsdParser::ParseFile() returns correct FPsdDocument with full layer tree including pixel data, text runs, groups, opacity, and visibility. Zero Python at plugin runtime.

2. **One-click PSD-to-WBP import with 15 widget types (Phase 3)** — IPsdLayerMapper / FLayerMappingRegistry pluggable dispatch maps image, text, group, button, progress, scroll, list, tile, overlay, switcher, and 5 more prefix types; FWidgetBlueprintGenerator produces a compilable, openable Widget Blueprint with correct anchors and z-order.

3. **Layer-Style stroke and drop shadow on text (Phase 4 + 4.1)** — FFontResolver, DPI conversion (0.75x), bold/italic typeface dispatch, AutoWrapText, plus a hand-rolled lfx2/FrFX raw-byte descriptor walker routing Photoshop Layer-Style Stroke and Drop Shadow to UTextBlock OutlineSettings and SetShadowOffset. All 5 text-effects fixture layers smoke-tested in-editor 2026-04-15.

4. **Full layer-effects pipeline with flatten fallback (Phase 5)** — SetRenderOpacity, Collapsed visibility, color-overlay tint, UImage drop-shadow duplicate, and bFlattenComplexEffects rasterization path for inner shadow / gradient overlay.

5. **Production editor workflow: preview dialog, reimport, CommonUI, animations (Phases 6-7)** — 9-slice Box-draw, Smart Object recursive child-WBP import, row/column anchor heuristics, SPsdImportPreviewDialog modal with layer checkboxes and type badges, non-destructive FReimportHandler, CommonUI UCommonButtonBase with IA binding, UWidgetAnimation generation from @show/@hide/@hover variants.

6. **Unified `@`-tag grammar replacing all ad-hoc prefix/suffix dispatch (Phase 9)** — FLayerTagParser::Parse produces FParsedLayerTags as single source of truth; 103 references across 22 files consume parsed struct; formal EBNF grammar spec, migration guide, and rewritten README layer-naming section shipped.

## Milestone Summary

**Decimal Phases:**
- Phase 4.1: Text Layer Effects Dispatch (inserted after Phase 9; gap-closure for TEXT-03/TEXT-04 surfaced by audit)

**Key Decisions:**
- PhotoshopAPI pre-built as static lib with /MD CRT (no CMake at UBT time)
- Trust PhotoshopAPI text API directly — no manual TySh parsing needed
- OpenImageIO ships from Phase 2 onward (needed for Smart Objects in Phase 6)
- ParseFile uses LayeredFile<bpp8_t>::read(filesystem::path); byte buffer spills to temp file (no in-memory stream overload)
- Text fill color from PhotoshopAPI is ARGB not RGBA — empirically discovered; lfx2 FrFX stroke color uses separate FLinearColor::FromSRGBColor path (no ARGB swizzle)
- AllMappers.h private header declares all 15 mapper classes (avoids ODR, keeps mappers internal)
- UWidgetBlueprintFactory::FactoryCreateNew for WBP creation; FAnchorCalculator suffix table ordered longest-first
- Invisible layers created as Collapsed (not skipped) to preserve layer count for runtime show/hide
- F9SliceImageLayerMapper at priority 150; FVariantsSuffixMapper at 200; bracket [L,T,R,B] stripped in TryParseSuffix
- FLayerTagParser as pure function with OutDiagnostics out-param; ParsedTags populated as post-pass after ConvertLayerRecursive
- Reimport identity-key via ParsedTags.CleanName (option-a, D-15 hard cutover); one-time orphaning of pre-Phase-9 WBPs; migration guide documents delete-and-reimport
- lfx2 discriminator at m_Data[4..7] (version=0x10), not [0..3] — confirmed empirically (8-byte prefix offset)
- DPI conversion for shadow offset applied in FTextLayerMapper (D-09 Option B) matching OutlineSize pattern

**Issues Resolved:**
- TEXT-03 gap: lfx2/FrFX descriptor walker hand-rolled to bypass PhotoshopAPI v0.9 black-box; raw PSD byte scan for 8BIMlfx2 signatures
- TEXT-04 gap: FPsdTextRun shadow fields re-activated; RouteTextEffects shadow branch with 0.75x DPI; P5 ExtractLayerEffects provides source data
- Forward scan direction bug (lfx2 before luni in layer record) — hotfix 792ef88
- Tag-* orphan reconciliation: REQUIREMENTS.md traceability had zero TAG rows (added TAG-01..09 all marked Complete)
- ROADMAP Phase 4/5 checkboxes stale — fixed in doc-hygiene commit 9bcde68

**Issues Deferred:**
- frameFXMulti VlLs stroke format (newer Photoshop) — safe bHasStroke=false fallback; post-v1
- Image-layer stroke rendering (D-12) — data populated, not consumed by image/group mappers; post-v1
- lrFX channel-order visual confirm for color overlay/shadow — host .uproject required; post-ship human UAT
- GitHub Actions CI — user decision D-04; post-v1
- CHANGELOG.md — user decision D-06; post-v1

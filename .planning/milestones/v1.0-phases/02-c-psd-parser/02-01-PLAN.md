<task type="auto">
  <name>Task 1: Vendor PhotoshopAPI libs and create ThirdParty external module</name>
  <files>Source/ThirdParty/PhotoshopAPI/PhotoshopAPI.Build.cs, Source/ThirdParty/PhotoshopAPI/Win64/include/.gitkeep, Source/ThirdParty/PhotoshopAPI/Win64/lib/.gitkeep, Source/ThirdParty/PhotoshopAPI/LICENSES/README.md, Scripts/build-photoshopapi.bat</files>
  <action>
    Create the ThirdParty layout per CONTEXT.md (PhotoshopAPI Vendoring decision):

    1. `Source/ThirdParty/PhotoshopAPI/PhotoshopAPI.Build.cs` — `ModuleType.External`. In the ctor:
       - Add `Path.Combine(ModuleDirectory, "Win64", "include")` and `Path.Combine(ModuleDirectory, "Win64", "include", "PhotoshopAPI")` to `PublicSystemIncludePaths` (final include subdir determined empirically from vendored tree).
       - For Win64 target, enumerate every `.lib` under `Win64/lib/` and add to `PublicAdditionalLibraries` via `Directory.GetFiles(libDir, "*.lib")`. This keeps the Build.cs resilient to the exact transitive set (openimageio, fmt, libdeflate, stduuid, eigen3, blosc2, zlib-ng, simdutf, PhotoshopAPI itself).
       - Guard with `if (Target.Platform == UnrealTargetPlatform.Win64)` — no Mac branch. Fail loudly via `throw new BuildException(...)` on other platforms.
       - Set `Type = ModuleType.External`, `PCHUsage = PCHUsageMode.NoSharedPCHs`.

    2. Create directory skeletons with `.gitkeep` files so the paths exist in git:
       - `Source/ThirdParty/PhotoshopAPI/Win64/include/.gitkeep`
       - `Source/ThirdParty/PhotoshopAPI/Win64/lib/.gitkeep`

    3. `Source/ThirdParty/PhotoshopAPI/LICENSES/README.md` — placeholder listing the licenses that must be committed alongside the vendored libs (PhotoshopAPI MIT, OpenImageIO BSD, fmt MIT, libdeflate MIT, stduuid MIT, Eigen MPL2, blosc2 BSD, zlib-ng zlib, simdutf Apache-2/MIT). User drops real LICENSE files here during bootstrap.

    4. `Scripts/build-photoshopapi.bat` — documented CMake bootstrap driver. Steps the script performs:
       - Clone PhotoshopAPI from upstream at a pinned tag into a temp dir
       - Run `vcpkg install` with triplet `x64-windows-static-md` (per CONTEXT.md — NOT `x64-windows-static`; UE uses /MD)
       - `cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE=<vcpkg>/scripts/buildsystems/vcpkg.cmake -DVCPKG_TARGET_TRIPLET=x64-windows-static-md -DPSAPI_BUILD_STATIC=ON`
       - `cmake --build build --config Release`
       - `robocopy` the resulting `.lib` files into `Source/ThirdParty/PhotoshopAPI/Win64/lib/` and public headers into `Source/ThirdParty/PhotoshopAPI/Win64/include/`
       - Echo a reminder to commit LICENSE files into `LICENSES/`.

    NOTE: The script is documentation/rebuild-from-source only. A fresh clone already has the vendored libs committed. The script does NOT run during normal UE builds.

    IMPORTANT: This task creates only the scaffolding. Actual `.lib` and `.h` drop-in happens as a checkpoint action by the user running `build-photoshopapi.bat` once (or manually dropping pre-built libs they already have). The plan's verify step confirms the build once those binaries are in place.
  </action>
  <verify>
    <automated>test -f Source/ThirdParty/PhotoshopAPI/PhotoshopAPI.Build.cs && test -f Scripts/build-photoshopapi.bat && test -d Source/ThirdParty/PhotoshopAPI/Win64/lib && test -d Source/ThirdParty/PhotoshopAPI/Win64/include</automated>
  </verify>
  <done>ThirdParty module exists, directory layout in place, bootstrap script documented, LICENSES placeholder committed.</done>

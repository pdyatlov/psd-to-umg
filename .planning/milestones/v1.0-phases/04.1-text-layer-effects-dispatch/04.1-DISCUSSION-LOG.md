## Stroke extraction + text precedence

### Q1: Stroke parse approach

| Option | Description | Selected |
|--------|-------------|----------|
| Add first-class stroke extraction | New FPsdLayerEffects fields (bHasStroke, StrokeColor, StrokeSize); parser decodes 'oglw' alongside sofi/dsdw; clean + future-proof | ✓ |
| Stroke as Text-only field | Parser routes stroke directly into Text.Outline* when parent is text; couples parser to layer-type awareness | |
| Keep as complex-flag + text-flatten skip | Leave classified under bHasComplexEffects; doesn't close TEXT-03 | |

### Q2: TySh vs Layer-Style precedence

| Option | Description | Selected |
|--------|-------------|----------|
| Layer-Style wins | Matches designer intent ("stroke the whole text block"); TySh usually residual | ✓ |
| TySh character-stroke wins | Phase 4 already wired TySh; apply Layer-Style only when TySh absent | |
| First non-zero wins, warn on conflict | UE_LOG warning when both present | |
| Sum sizes, average colors | Usually undesirable — produces thicker stroke than either alone | |

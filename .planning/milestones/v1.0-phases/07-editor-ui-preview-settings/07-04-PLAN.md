Create FCommonUIButtonLayerMapper (header + cpp):

**Header** — derives from IPsdLayerMapper:
- GetPriority() returns 95 (higher priority than FButtonLayerMapper at 100, so it matches first when active)
- CanMap(): returns true only if `UPSD2UMGSettings::Get()->bUseCommonUI == true` AND layer name starts with "Button_"
- Map(): creates UCommonButtonBase, handles child state layers (Normal, Hovered, Pressed, Disabled) same as FButtonLayerMapper, then parses input action binding
- GetWidgetTypeName() returns "CommonButtonBase"

**Cpp implementation:**

CanMap():
```cpp
if (!UPSD2UMGSettings::Get()->bUseCommonUI) return false;
return Layer.Name.StartsWith(TEXT("Button_"), ESearchCase::IgnoreCase);
```

Map():
1. Check `FModuleManager::Get().IsModuleLoaded(TEXT("CommonUI"))`. If not loaded:
   - `UE_LOG(LogPSD2UMG, Error, TEXT("PSD2UMG: bUseCommonUI is enabled but the CommonUI module is not loaded. Falling back to UButton."));`
   - return nullptr (registry falls through to FButtonLayerMapper)
2. `UCommonButtonBase* Btn = Tree->ConstructWidget<UCommonButtonBase>(UCommonButtonBase::StaticClass(), FName(*CleanName));`
3. Handle child state brushes same as FButtonLayerMapper (copy the pattern from FButtonLayerMapper::Map)
4. Parse input action from layer name: regex or string search for `[IA_...]` bracket suffix
   - Extract text between `[` and `]`, e.g. "Button_Confirm[IA_Confirm]" -> "IA_Confirm"
   - Search asset registry for UInputAction matching that name under `UPSD2UMGSettings::Get()->InputActionSearchPath.Path`
   - If found: `Btn->SetTriggeringEnhancedInputAction(IA);`
   - If not found: `UE_LOG(LogPSD2UMG, Warning, TEXT("PSD2UMG: Input action asset '%s' not found at '%s'. Button created without binding."), *IAName, *SearchPath);`

Include `CommonUI/Public/CommonButtonBase.h` and `EnhancedInput/Public/InputAction.h`.

**Registry registration:** In AllMappers.h, add `#include "FCommonUIButtonLayerMapper.h"`. In FLayerMappingRegistry constructor/init, register `MakeShared<FCommonUIButtonLayerMapper>()`. Since priority 95 < 100 (FButtonLayerMapper), it will be checked first.

Wait — check the priority convention. If lower number = higher priority, set to 95. If higher number = higher priority, set to 105. Read FLayerMappingRegistry to confirm sort order.

Create FPsdWidgetAnimationBuilder as a static helper class:

**Header:**
```cpp
#pragma once
#include "CoreMinimal.h"

class UWidgetBlueprint;
class UWidgetAnimation;

class FPsdWidgetAnimationBuilder
{
public:
    // Create opacity fade animation (for _show and _hide variants)
    // FromOpacity/ToOpacity: 0.0-1.0, DurationSec in seconds
    static UWidgetAnimation* CreateOpacityFade(
        UWidgetBlueprint* WBP,
        const FString& AnimName,
        const FName& TargetWidgetName,
        float FromOpacity, float ToOpacity, float DurationSec);

    // Create scale animation (for _hover variant)
    // FromScale/ToScale as FVector2D
    static UWidgetAnimation* CreateScaleAnim(
        UWidgetBlueprint* WBP,
        const FString& AnimName,
        const FName& TargetWidgetName,
        FVector2D FromScale, FVector2D ToScale, float DurationSec);

    // Detect animation variants in layer name suffixes and create animations
    // Checks for _show, _hide, _hover suffixes; creates appropriate animations
    // Call after widget tree is built
    static void ProcessAnimationVariants(
        UWidgetBlueprint* WBP,
        const FName& WidgetName,
        const FString& LayerName);
};
```

**Cpp implementation:**

CreateOpacityFade() — per RESEARCH.md Pattern 5:
- Create UWidgetAnimation with name AnimName, outer = WBP
- Create UMovieScene, set tick resolution 24000/1, display rate 30/1
- Set playback range 0 to DurationSec * 24000
- AddPossessable for target widget name
- AddTrack<UMovieSceneFloatTrack>, set property "RenderOpacity"
- CreateNewSection, set range, GetChannel, AddLinearKey at frame 0 and end frame
- Create FWidgetAnimationBinding with WidgetName, AnimationGuid
- Add to WBP->Animations
- Do NOT call BindPossessableObject (per RESEARCH anti-pattern warning)

CreateScaleAnim():
- Same UMovieScene setup
- Two float tracks: property "RenderTransform.Scale.X" and "RenderTransform.Scale.Y"
- Keyframe from FromScale.X/Y to ToScale.X/Y

ProcessAnimationVariants():
- If LayerName ends with "_show": CreateOpacityFade(WBP, LayerName+"_Show", WidgetName, 0.0f, 1.0f, 0.3f)
- If LayerName ends with "_hide": CreateOpacityFade(WBP, LayerName+"_Hide", WidgetName, 1.0f, 0.0f, 0.3f)
- If LayerName ends with "_hover": CreateScaleAnim(WBP, LayerName+"_Hover", WidgetName, FVector2D(1.0f), FVector2D(1.05f), 0.15f)

Include: MovieScene.h, MovieSceneFloatTrack.h, MovieSceneFloatSection.h, MovieSceneFloatChannel.h, WidgetAnimation.h, WidgetBlueprint.h.

CUI-04 (ScrollBox content height): no code needed — per D-12, UMG handles at runtime. Just ensure existing ScrollBox mapper places children inside the ScrollBox (already done in Phase 3).

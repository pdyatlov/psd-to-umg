# 09-02.1 Hotfix ŌĆö Remove DetectHorizontalRow/DetectVerticalColumn auto-heuristic

## What happened

Plan 09-02 did not deliver a DESIGN truth: auto-detection of HBox/VBox was supposed to be **removed entirely**, not merely guarded.

- `DESIGN.md:7` ŌĆö *"Remove auto-detection heuristics (e.g. `DetectHorizontalRow`) ŌĆö behavior must be explicit."*
- `DESIGN.md:104` ŌĆö *"`FWidgetBlueprintGenerator::DetectHorizontalRow` removed. No auto-HBox wrapping; groups without `[hbox]` stay `CanvasPanel`."*

What 09-02-PLAN.md actually specified (lines 235ŌĆō255, 260) was an **R-05 guard**: skip children with explicit `@anchor:stretch-*` from the auto-row/column heuristic. The heuristic itself was left in place. 09-02-SUMMARY.md signed off on the guard. Plan 09-03 started on top of a phase that didn't match its DESIGN contract.

## User-visible symptom

Reimporting a PSD where a group named `Buttons` (no tag) contains horizontally-aligned buttons produced `Root_Canvas ŌåÆ Buttons ŌåÆ UHorizontalBox`. Per DESIGN, untagged groups must stay `UCanvasPanel`; only `@hbox`/`@vbox` tags create box layouts (handled by `FHBoxLayerMapper` / `FVBoxLayerMapper` at priority 200).

## Fix

**`Source/PSD2UMG/Private/Generator/FWidgetBlueprintGenerator.cpp`**
- Deleted `AlignmentTolerancePx`, `IsExplicitStretchAnchor`, `AnyChildHasExplicitStretch`, `DetectHorizontalRow`, `DetectVerticalColumn`.
- Deleted the auto-wrap block in `PopulateCanvas` (the `if (Layers.Num() >= 2 && !AnyChildHasExplicitStretch(...))` branch creating UHorizontalBox/UVerticalBox).
- Removed now-unused includes: `Components/HorizontalBox.h`, `Components/HorizontalBoxSlot.h`, `Components/VerticalBox.h`, `Components/VerticalBoxSlot.h`.

**`Source/PSD2UMG/Tests/FWidgetBlueprintGenSpec.cpp`**
- R-05 spec repurposed: previously asserted "explicit `@anchor:stretch-h` suppresses auto-row heuristic"; now asserts "untagged group with horizontally-aligned children stays `CanvasPanel`". Same expected outputs (`HBoxCount == 0`, `ImageCount == 3`), new rationale. Removed `@anchor:stretch-h` from the middle layer's name ŌĆö no longer part of the invariant under test.

## Process lesson (for future phases with a DESIGN contract)

- `gsd-plan-checker` and `gsd-verifier` ran goal-backward from **PLAN must_haves**, not from **DESIGN truths**. When DESIGN contains discrete contractual statements the PLAN must inherit, the checker should verify `PLAN.must_haves.truths ŌŖć DESIGN truth statements` before planning is accepted.
- R-05 was documented as a *guard*, not as *removal*. The plan author transformed a "remove X" DESIGN item into an "add guard on X" PLAN item, which passed all downstream checks because the checks only saw the PLAN.

Recommend: add a verify-against-DESIGN step to `gsd:plan-phase` when a `DESIGN.md` exists in the phase directory.

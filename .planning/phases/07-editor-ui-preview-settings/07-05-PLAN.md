Add a new static method to FWidgetBlueprintGenerator:

```cpp
// Update an existing Widget Blueprint from a re-parsed PSD document.
// - Matches existing widgets by name (D-05)
// - Updates PSD-sourced properties: position, size, image data, text (D-06)
// - Preserves manual edits: Blueprint logic, bindings, animations, manually-added widgets (D-06)
// - Widgets from deleted PSD layers are kept as orphans (D-07)
// - SkippedLayerNames: layers the user unchecked in the preview dialog
static bool Update(
    UWidgetBlueprint* ExistingWBP,
    const FPsdDocument& NewDoc,
    const TSet<FString>& SkippedLayerNames);
```

Implementation of Update():
1. Get the existing WidgetTree from ExistingWBP->WidgetTree
2. Build a TMap<FString, UWidget*> of existing widgets by their GetName()
3. Walk NewDoc.RootLayers recursively. For each layer:
   a. If layer name is in SkippedLayerNames, skip
   b. Look up existing widget by name in the map
   c. If found (existing): update PSD-sourced properties only:
      - CanvasPanelSlot position/size from layer bounds
      - For UImage: re-import texture, update brush
      - For UTextBlock: update text content, font size, color
      - Remove from the map (marks as "handled")
   d. If not found (new layer): create new widget via mapper, add to widget tree
4. Widgets remaining in the map after traversal = orphans from deleted PSD layers. Per D-07, keep them (do nothing).
5. Compile and save the blueprint (same as end of Generate()).
6. Return true on success.

For change detection helper (used by reimport handler to annotate tree items):
Add a static helper:
```cpp
static EPsdChangeAnnotation DetectChange(
    const FPsdLayer& NewLayer,
    UWidget* ExistingWidget); // returns New, Changed, or Unchanged
```
DetectChange compares: position/size (bounds vs slot), text content, opacity. If any differ -> Changed. If no existing widget -> New. Otherwise -> Unchanged.

Create FPsdReimportHandler (header + cpp):

**Header:**
```cpp
#pragma once
#include "EditorReimportHandler.h"

class FPsdReimportHandler : public FReimportHandler
{
public:
    virtual bool CanReimport(UObject* Obj, TArray<FString>& OutFilenames) override;
    virtual void SetReimportPaths(UObject* Obj, const TArray<FString>& NewReimportPaths) override;
    virtual EReimportResult::Type Reimport(UObject* Obj) override;
};
```

**Cpp:**

CanReimport():
- Cast Obj to UWidgetBlueprint. If null, return false.
- Read PSD2UMG.SourcePsdPath from package metadata. If empty, return false.
- Add path to OutFilenames, return true.

SetReimportPaths():
- Cast to UWidgetBlueprint, update PSD2UMG.SourcePsdPath metadata with NewReimportPaths[0].

Reimport():
1. Cast Obj to UWidgetBlueprint. Read source PSD path from metadata.
2. Check file exists. If not, return EReimportResult::Failed with log error.
3. Parse PSD via PSD2UMG::Parser::ParseFile(SourcePath, Doc, Diag).
4. Build tree items via SPsdImportPreviewDialog::BuildTreeFromDocument(Doc).
5. Annotate tree items with change status: for each item, find matching widget in existing WBP by name, call FWidgetBlueprintGenerator::DetectChange() to set EPsdChangeAnnotation.
6. Show preview dialog (modal) with bIsReimport=true. Dialog shows [new]/[changed]/[unchanged] annotations.
7. On confirm: collect unchecked layer names into TSet<FString>, call FWidgetBlueprintGenerator::Update(WBP, Doc, SkippedNames).
8. On cancel: return EReimportResult::Cancelled.
9. Update PSD2UMG.SourcePsdPath metadata (in case path changed).
10. Return EReimportResult::Succeeded.

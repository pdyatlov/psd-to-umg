# Phase 7 — UI Design Contract

> Visual and interaction contract for Phase 7: Editor UI, Preview & Settings.
> Generated by gsd-ui-researcher.
>
> **IMPORTANT — Design System Context:**
> This is Unreal Engine 5 Slate editor UI, NOT web/game UI. The design system is
> FAppStyle (formerly FEditorStyle). All spacing, color, and typography values are
> expressed as Slate margin/padding values and FAppStyle style names, not CSS/Tailwind.
> No shadcn, no Tailwind, no web component library applies.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Slate / FAppStyle — built into UE5 editor) |
| Preset | not applicable |
| Component library | Unreal Engine Slate (SWindow, STreeView, SCheckBox, SButton, STextBlock, SEditableTextBox, SBorder, SSplitter, SScrollBox, SHeaderRow) |
| Icon library | FAppStyle::GetBrush() — use existing editor icon set (e.g. "Icons.Import", "Icons.Refresh", "Icons.FolderOpen", "ContentBrowser.AssetActions") |
| Font | FAppStyle::GetFontStyle("NormalFont") / "SmallFont" / "BoldFont" — engine-managed, no custom fonts |

**Design System Note:** All widget construction uses `SNew(SWidgetType)` Slate macros.
Colors are sourced from `FAppStyle::GetSlateColor()`. The editor dark theme is the
baseline — do not hard-code hex values for structural colors. Use named style tokens only.

---

## Spacing Scale

Slate uses `FMargin(horizontal, vertical)` or `FMargin(left, top, right, bottom)`.
All values in unscaled pixels (Slate DPI-scales automatically).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 2px | Icon-to-label inline gap, badge inner padding |
| sm | 4px | Row inner padding (checkbox-to-text gap), compact element spacing |
| md | 8px | Section inner padding, dialog content padding sides |
| lg | 12px | Row height padding (top+bottom combined), button padding |
| xl | 16px | Dialog section separator, group header padding |
| 2xl | 24px | Dialog outer content margin |
| 3xl | 32px | Minimum dialog width guard (not spacing — used for min-size) |

Exceptions:
- Tree row height: fixed 20px (matches STreeView standard row height in UE editor)
- Dialog minimum size: 480px wide × 520px tall (enforced via SWindow::SetSizingRule + MinWidth/MinHeight)
- Bottom button bar height: 40px (matches standard UE editor dialog button bars)
- Output path row: full-width minus 28px for browse button (SHorizontalBox fill + fixed slot)

---

## Typography

All fonts use FAppStyle style names. Do NOT construct FSlateFontInfo with custom sizes —
use the named styles from the running editor theme.

| Role | FAppStyle Style Name | Usage |
|------|---------------------|-------|
| Body | `"NormalFont"` | Layer names in tree rows, settings property labels, status messages |
| Label/Small | `"SmallFont"` | Widget type badge text, change annotation labels ([new], [changed], [unchanged]), secondary descriptive text |
| Heading | `"NormalFontBold"` (or `"BoldFont"`) | Dialog section headers ("Layer Tree", "Output Settings"), dialog title bar |
| Monospace | `"NormalFont"` (standard) | Output path field (no special monospace needed; FDirectoryPath field is standard) |

Line height: Slate computes line height from font metrics automatically. No override needed.
Font weight: Expressed via style name selection (NormalFont vs BoldFont), not weight integer.

## Color

Slate editor UI is theme-driven. Use named style tokens; never hard-code hex.
The mapping below shows the *semantic intent* and the correct FAppStyle call.

| Role | FAppStyle Token / Call | Usage |
|------|----------------------|-------|
| Dominant (60%) — dark panel background | `FAppStyle::GetColor("WindowBackground")` or `FAppStyle::Get().GetWidgetStyle<FTableRowStyle>("TableView.Row").EvenRowBackgroundBrush` | Dialog background, tree area background |
| Secondary (30%) — slightly lighter surface | `FAppStyle::GetBrush("ToolPanel.GroupBorder")` on SBorder | Section containers, output path row background, button bar background |
| Accent (10%) — UE editor blue/gold highlight | `FAppStyle::GetSlateColor("SelectionColor")` | Selected tree rows only; no other use |
| Badge: UButton | Hardcoded Slate color: `FLinearColor(0.2f, 0.5f, 0.9f, 1.f)` — blue | Widget type badge background for Button_ layers |
| Badge: UImage | `FLinearColor(0.3f, 0.65f, 0.3f, 1.f)` — green | Widget type badge background for Image layers |
| Badge: UTextBlock | `FLinearColor(0.8f, 0.6f, 0.1f, 1.f)` — amber | Widget type badge background for Text layers |
| Badge: UCanvasPanel / group | `FLinearColor(0.5f, 0.5f, 0.5f, 1.f)` — grey | Widget type badge background for group/canvas layers |
| Badge: UCommonButtonBase | `FLinearColor(0.55f, 0.2f, 0.8f, 1.f)` — purple | Widget type badge when CommonUI mode is active |
| Badge: Other widget types | `FLinearColor(0.4f, 0.4f, 0.6f, 1.f)` — slate-blue | All remaining widget type badges |
| Change annotation: [new] | `FLinearColor(0.3f, 0.75f, 0.3f, 1.f)` — green text | Reimport preview — new layer indicator |
| Change annotation: [changed] | `FLinearColor(0.9f, 0.65f, 0.1f, 1.f)` — amber text | Reimport preview — modified layer indicator |
| Change annotation: [unchanged] | `FAppStyle::GetSlateColor("DefaultForeground")` at 50% opacity | Reimport preview — unchanged layer indicator |
| Destructive | `FAppStyle::GetSlateColor("Colors.AccentRed")` | None in this phase — no destructive actions |

Accent (selection highlight) reserved for: selected tree row highlight only.
Badge colors are the ONLY hard-coded color values permitted in this phase — they are semantic
type indicators with no FAppStyle equivalent.

## Component Inventory

### 1. Import Preview Dialog (`SPsdImportPreviewDialog`)

**Container:** `SWindow` (modal, `SizingRule::UserSized`, MinWidth=480, MinHeight=520, default 600×640)
**Layout:** `SVerticalBox` root with three slots:

#### Slot A — Layer Tree (fills available height)
- `SBorder` with `"ToolPanel.GroupBorder"` → contains `SScrollBox` → contains `STreeView<TSharedPtr<FPsdLayerTreeItem>>`
- Tree columns (manual layout per row, NOT SHeaderRow — simpler for single-level):
  - Col 1: `SCheckBox` (16×16) — layer include/exclude toggle. Default: checked. Unchecked = layer skipped on import.
  - Col 2: Indentation spacer (8px per depth level, max 5 levels = 40px)
  - Col 3: Layer name `STextBlock` with `"NormalFont"`, fills remaining width
  - Col 4: Widget type badge (see Badge spec below), right-aligned, fixed width 120px
  - Col 5 (reimport only): Change annotation label, right-aligned, fixed width 80px

**Badge spec (`SPsdWidgetTypeBadge`):**
- `SBorder` with `FMargin(4, 2)` padding, 6px border radius (via `RoundedBoxBrush` or custom brush)
- Background: badge color from color table above
- Text: `STextBlock` with `"SmallFont"`, white (`FLinearColor::White`), widget class name without `U` prefix (e.g. "Button", "Image", "TextBlock", "CanvasPanel")
- Fixed width: 120px container with text center-aligned

#### Slot B — Output Path Row (fixed height 36px)
- `SHorizontalBox`:
  - Label: `STextBlock` `"Output Path:"` `"NormalFont"` fixed width 88px
  - Path field: `SEditableTextBox` fills remaining width minus 32px, pre-filled from `UPSD2UMGSettings::WidgetBlueprintAssetDir`
  - Browse button: `SButton` 28×28 with `FAppStyle::GetBrush("Icons.FolderOpen")` icon, no label text
- Padding: `FMargin(8, 4)` on row container

#### Slot C — Button Bar (fixed height 40px)
- `SBorder` with `"ToolPanel.GroupBorder"` background
- `SHorizontalBox` right-aligned (fill spacer on left):
  - Cancel button: `SButton` label `"Cancel"`, `"NormalFont"`, standard Slate button style
  - Import button: `SButton` label `"Import"` (fresh import) or `"Apply Reimport"` (reimport), primary style via `"PrimaryButton"` FAppStyle style if available, else standard button with `"NormalFont"` BoldFont

### 2. Plugin Settings Panel (extends `UPSD2UMGSettings`)

Rendered automatically by UE `UDeveloperSettings` detail panel infrastructure.
No custom Slate widget needed — all fields use `UPROPERTY` category grouping.

**Category grouping** (use `Category = "PSD2UMG|{Group}"` syntax):

| Group name | Properties |
|------------|-----------|
| `"PSD2UMG|General"` | `bEnabled`, `bShowPreviewDialog` |
| `"PSD2UMG|Output"` | `WidgetBlueprintAssetDir`, `TextureAssetDir`, `TextureSrcDir` |
| `"PSD2UMG|Typography"` | `DefaultFont`, `FontMap`, `SourceDPI` |
| `"PSD2UMG|Effects"` | `bFlattenComplexEffects`, `MaxSmartObjectDepth` |
| `"PSD2UMG|CommonUI"` | `bUseCommonUI`, `InputActionSearchPath` |

New properties to add (source: CONTEXT.md D-13):
- `bShowPreviewDialog` — `bool`, default `true`, `"PSD2UMG|General"`, tooltip: `"Show layer preview dialog before every import. Disable for batch import workflows."`
- `bUseCommonUI` — `bool`, default `false`, `"PSD2UMG|CommonUI"`, tooltip: `"When enabled, Button_ layers generate UCommonButtonBase instead of UButton. Requires CommonUI plugin."`
- `InputActionSearchPath` — `FDirectoryPath`, default `/Game/Input/`, `"PSD2UMG|CommonUI"`, tooltip: `"Search path for UInputAction assets referenced in layer names (e.g. Button_Confirm[IA_Confirm])."`
- `SourceDPI` — `int32`, default `72`, `"PSD2UMG|Typography"`, tooltip: `"Photoshop source document DPI. Used for pt→px conversion. Default 72 = 1pt equals 1px in UMG."`

### 3. Content Browser Context Menu

Registered via `FContentBrowserMenuExtender_SelectedAssets`.
No custom visual widget — entries appear as standard `FUIAction`-backed menu items in the
existing UE Content Browser right-click menu.

**Menu entries:**
- On PSD texture assets (`.psd` source): `"Import as Widget Blueprint"` — icon `"Icons.Import"` from FAppStyle
- On Widget Blueprint assets (with PSD metadata tag): `"Reimport from PSD"` — icon `"Icons.Refresh"` from FAppStyle

Both entries placed in a dedicated `"PSD2UMG"` sub-menu section with separator above it.
Section label: `"PSD2UMG"` in `"SmallFont"`.

### 4. Reimport Preview (reuses `SPsdImportPreviewDialog`)

Same component as import preview. Differences:
- Window title: `"Reimport from PSD — {WBP Name}"`
- Change annotation column (Col 5) is VISIBLE (hidden on first import)
- Change annotation values: `"[new]"` / `"[changed]"` / `"[unchanged]"` in SmallFont, colored per color table
- Bottom bar button label: `"Apply Reimport"` instead of `"Import"`
- Unchecked layers with `[changed]` annotation retain existing widget (no update applied)
- Unchecked layers with `[new]` annotation are not created

## Interaction Contracts

### Tree Checkbox Behavior
- Default state: ALL rows checked
- Unchecking a parent (group layer) automatically unchecks all children (cascade)
- Re-checking a parent restores all children to checked
- Visual: indeterminate state (`ECheckBoxState::Undetermined`) shown on parent when some children are unchecked
- Keyboard: Space bar toggles focused row checkbox

### Dialog Open/Close
- Dialog opens modally before any asset creation begins (CONTEXT.md D-01)
- "Cancel" button closes dialog; import aborts; no assets created
- "Import" / "Apply Reimport" button closes dialog and proceeds with generation using checked layers only
- Closing window via X button = Cancel behavior
- Dialog remembers last used output path for the session (not persisted to config)

### Output Path Browse Button
- Opens `IDesktopPlatform::OpenDirectoryDialog` (standard UE folder picker)
- Returned path written to the `SEditableTextBox` field
- Path validated on Import click: must start with `/Game/` — if invalid, show inline `STextBlock` error in red (`FLinearColor(0.9f, 0.2f, 0.2f, 1.f)`) below the path field

### Tree Expansion
- Groups (folders) are expanded by default on dialog open (all levels)
- User can collapse/expand via standard STreeView arrow toggle
- Tree is read-only for structure; only checkboxes are interactive

### Settings Property Tooltips
- Every new UPROPERTY must have a non-empty `ToolTip` meta field (see Component Inventory above)
- Existing properties without tooltips: add tooltip on first edit (editorial improvement, not blocking)

## Animation Contract (CUI-03)

Animation generation uses Slate-side UWidgetAnimation creation. The generated animations
are stored on the Widget Blueprint asset, not in the dialog.

| Variant suffix | Animation name | Type | Duration | From | To |
|---------------|---------------|------|----------|------|----|
| `_show` | `"{LayerName}_Show"` | Opacity fade | 0.3s | 0.0 | 1.0 |
| `_hide` | `"{LayerName}_Hide"` | Opacity fade | 0.3s | 1.0 | 0.0 |
| `_hover` | `"{LayerName}_Hover"` | RenderScale | 0.15s | (1.0, 1.0) | (1.05, 1.05) |

Animation easing: linear (no easing curve — simpler to generate programmatically).
Source: CONTEXT.md D-11.

## Slate Implementation Notes

These notes are binding for the executor — they are design decisions, not suggestions.

### SWindow Construction
```cpp
// Modal dialog pattern — use this exact SizingRule
SNew(SWindow)
    .Title(LOCTEXT("ImportPreviewTitle", "PSD2UMG — Import Preview"))
    .SizingRule(ESizingRule::UserSized)
    .AutoCenter(EAutoCenter::PrimaryWorkArea)
    .ClientSize(FVector2D(600, 640))
    .MinWidth(480)
    .MinHeight(520)
    .SupportsMinimize(false)
    .SupportsMaximize(false)
```

### STreeView Row Generation
```cpp
// Row widget builder — implement in GenerateRow delegate
SNew(STableRow<TSharedPtr<FPsdLayerTreeItem>>, OwnerTable)
[
    SNew(SHorizontalBox)
    + SHorizontalBox::Slot().AutoWidth().Padding(4, 0)
    [ SNew(SCheckBox)... ]
    + SHorizontalBox::Slot().AutoWidth()
    [ SNew(SSpacer).Size(FVector2D(IndentPx, 1)) ]
    + SHorizontalBox::Slot().FillWidth(1.f).VAlign(VAlign_Center)
    [ SNew(STextBlock).Font(FAppStyle::GetFontStyle("NormalFont"))... ]
    + SHorizontalBox::Slot().AutoWidth().HAlign(HAlign_Right)
    [ /* Badge widget 120px */ ]
    + SHorizontalBox::Slot().AutoWidth().HAlign(HAlign_Right)
    [ /* Change annotation — hidden on first import */ ]
]
```

### FAppStyle Icon Names (verified UE5 names)
- Folder browse: `"Icons.FolderOpen"` or `"ContentBrowser.AssetActions.OpenInExternalEditor"`
- Import: `"Icons.Import"` or `"AssetEditor.Import"`
- Refresh/reimport: `"Icons.Refresh"`

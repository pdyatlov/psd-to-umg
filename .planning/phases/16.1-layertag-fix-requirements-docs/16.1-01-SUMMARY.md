## Decisions Made

- D-02 switch maps Gradient/SolidFill/Shape to EPsdTagType::Image (same target as EPsdLayerType::Image). The three mappers (FGradientLayerMapper, FSolidFillLayerMapper, FShapeLayerMapper) all already gate on `Type==Image` so no mapper changes needed.
- bEffectivelyVisible added as a computed visibility flag propagated top-down in BuildTreeRecursive. This avoids per-row parent traversal and ensures the ColorAndOpacity ternary receives the correct effective visibility for subtrees under a hidden parent.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Propagate bEffectivelyVisible to children in BuildTreeRecursive**

- **Found during:** Task 3 / human checkpoint review (Task 4)
- **Issue:** The original HIDDEN-02 plan dimmed only the directly-hidden layer (Item->bLayerVisible). Children of a hidden group remained at full opacity even though their parent group is hidden in the PSD. The user identified this during the Task 4 checkpoint visual review.
- **Fix:** Added bEffectivelyVisible field to FPsdLayerTreeItem (computed in BuildTreeRecursive as `parentEffectivelyVisible && bLayerVisible`). The ColorAndOpacity ternary in SPsdImportPreviewDialog uses `Item->bEffectivelyVisible` so the entire subtree under a hidden group dims correctly.
- **Files modified:** Source/PSD2UMG/Private/Reimport/FPsdReimportHandler.cpp, Source/PSD2UMG/Private/Reimport/FPsdReimportHandler.h
- **Verification:** Human visual approval during Task 4 checkpoint
- **Committed in:** `79579ff` (extra commit between Task 3 and Task 4 approval)

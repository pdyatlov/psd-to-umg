## Decisions Made

- Two-compile sequence: animations must land in `WBP->Animations` before compile #1 (so UPROPERTIES exist for K2 `UK2Node_VariableGet`); K2 nodes must be created after compile #1 (so `SkeletonGeneratedClass` has the `UButton FObjectProperty`); compile #2 bakes the K2 graph into `GeneratedClass`.
- Disabled state (D-02): `UButton` has no `OnDisabled` delegate. No `{CleanName}_Disabled` animation created, no K2 injection attempted. Comments avoid using "OnDisabled" string to keep acceptance grep clean.
- Non-destructive reimport: `Update()` refreshes animations additively but does not re-inject K2 wiring, so designer modifications to the Event Graph survive reimport.
- Comment phrasing for "OnDisabled": rephrased as "UButton exposes no delegate for the disabled state" to prevent false hits on acceptance criterion `grep -c "OnDisabled"` returns 0.

## Root causes fixed during Plan 04

### Bug 1: MovieScene naming mismatch (animation variable null in PIE)
`BindAnimationsStatic` (WidgetBlueprintGeneratedClass.cpp) matches animation UPROPERTYs by
`Animation->GetMovieScene()->GetFName()`. Our `CreateMovieScene` used `MakeUniqueObjectName`
which produced "MovieScene_0" while the UPROPERTY was named "MyBtn_Hover" — silent mismatch,
animation variable stayed null, `PlayAnimation(nullptr)` was a no-op.
Fix: parent MovieScene to the animation, name it `Anim->GetFName()` (matches UMG editor pattern
in AnimationTabSummoner.cpp:640).

### Bug 2: ReverseAnimation silent no-op after animation completes
`ReverseAnimation` calls `GetAnimationState` which searches `ActiveAnimations`. After
`PlayAnimation` completes one loop, `OnStopped` sets `bIsPendingDelete=true` and the state is
removed from `ActiveAnimations` on the next tick. A user hovering longer than the animation
duration (0.15s) then unhovering got a silent no-op — button stuck at hover color.
Fix: replaced all `ReverseAnimation` K2 nodes with `PlayAnimation(StartAtTime=DurationSec,
PlayMode=Reverse=1)`, which always creates a fresh animation state.

## Cross-Phase Integration

**15/17 integration paths confirmed wired.** Two issues found:

### Issue 1 — Mapper Priority Collision (Medium, GRAD-01/SHAPE-01)

`FImageLayerMapper`, `FFillLayerMapper`, `FSolidFillLayerMapper`, and `FShapeLayerMapper` all register at priority 100. After Phase 16.1's D-02 switch fix, `FImageLayerMapper::CanMap` (gates on `ParsedTags.Type == EPsdTagType::Image`) now returns `true` for any untagged gradient, solid-fill, or shape layer — overlapping the type-based mappers.

`TArray::Sort` with equal comparator values is not stable. If `FImageLayerMapper` sorts before `FSolidFillLayerMapper`/`FShapeLayerMapper`, those layers' `Map()` calls `FTextureImporter::ImportLayer` on empty `RGBAPixels` → returns nullptr → layer dropped silently.

**In practice:** Insertion order happens to be preserved by current runtime sort. Tests pass. But this is implementation-defined behavior.

**Fix:** Raise `FFillLayerMapper`, `FSolidFillLayerMapper`, `FShapeLayerMapper` to priority 101, or narrow `FImageLayerMapper::CanMap` to exclude `Layer.Type == Gradient/SolidFill/Shape`.

### Issue 2 — Reimport Cache Leak (Low, FONT-01)

`FPsdReimportHandler::Reimport` calls `FWidgetBlueprintGenerator::Update` but does NOT call `FFontResolver::InvalidateDiscoveryCache()`. The discovery cache (`GDiscoveryCache`) persists across reimport sessions. Fonts added to the project between reimports are invisible to the AutoDiscovered resolver.

**Fix:** Add `#include "Mapper/FontResolver.h"` and `PSD2UMG::FFontResolver::InvalidateDiscoveryCache()` call to `FPsdReimportHandler.cpp` after the `FWidgetBlueprintGenerator::Update` call.
